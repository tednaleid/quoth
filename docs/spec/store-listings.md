## Detailed description

```
Quoth turns the captions on any YouTube video into a clean, formatted transcript that lives next to the player. Click any word to jump the video to that moment. The active passage scrolls with playback so you can follow along without losing your place.

Everything runs in your browser. No account, no server, no analytics. Quoth fetches captions through YouTube's own API and formats them locally; nothing about what you watch leaves your machine.

Features:
- Side panel transcript synced to the video's playback position
- Pop out the transcript into its own tab for full-width reading
- Click any word to seek the video to that timestamp
- Paragraph segmentation for readability
- Local cache so revisited videos load instantly
- Works on any YouTube video that has captions

Permissions are scoped to youtube.com only.
```

## Permission justifications

Used in the Chrome Web Store "Permissions justification" section and in
the AMO "Reasons for permissions" if requested. Cross-checked against
`wxt.config.ts` and the README permissions table.

| Permission | Justification |
|---|---|
| `host_permissions: *://*.youtube.com/*` | Content scripts run on YouTube watch pages to read playback state and fetch captions through YouTube's Innertube API. No other site is accessed. |
| `activeTab` | Identifies the currently active YouTube tab so the side panel can connect to the right video. |
| `tabs` | Queries open tabs to find the YouTube watch page and routes messages between the side panel and the content script. |
| `storage` | Caches parsed transcripts locally (keyed by video ID) so repeat visits load instantly. |
| `unlimitedStorage` | Removes the 5 MB default quota. Single transcripts are small (~200 KB) but heavy users benefit from a larger cache. |
| `contextMenus` | Adds a right-click entry to open the transcript in a popout tab. |
| `sidePanel` (Chrome only) | Renders the transcript in Chrome's native side panel. Firefox uses `sidebar_action` and does not need this permission. |

## Source-code submission (AMO)

AMO reviewers can request the source archive when they see minified
output. Provide:

- Public repository URL: https://github.com/tednaleid/quoth
- Build instructions (paste into AMO's source-code field):

```
Build tools (any one):
- Preferred: bun (https://bun.sh) -- install with `curl -fsSL https://bun.sh/install | bash`
- Fallback: Node.js 20+ with npm (already installed on most review machines)

Build with bun (deterministic, matches the submitted .xpi):
  bun install --frozen-lockfile
  bunx wxt build --browser firefox

Build with npm (no lockfile, minor version drift possible):
  npm install
  npx wxt build --browser firefox

In both cases the output appears at .output/firefox-mv2/ and matches the
contents of the submitted .xpi after Mozilla's signing wrapper.

Repository: https://github.com/tednaleid/quoth
```

## Reviewer notes (AMO "Notes for reviewers" / Chrome "Notes for the reviewer")

Paste into the reviewer-notes field at submission time. Address validator
warnings up front so reviewers don't have to ask. Bump the version number
in the build section before each submission.

```
Hi reviewer, thanks for taking a look. A few notes:

Build & reproduction
- Source archive uploaded matches this version's package.json. Build with bun (preferred): `bun install --frozen-lockfile && bunx wxt build --browser firefox`. Or with npm: `npm install && npx wxt build --browser firefox`. Output at .output/firefox-mv2/ matches the submitted .xpi after Mozilla's signing.
- Repository: https://github.com/tednaleid/quoth

Validator warnings (all safe)
1. "Unsafe assignment to innerHTML" in chunks/TranscriptView-*.js: this is Svelte 5 framework runtime, not application code. The compiled function uses a Trusted Types policy named "svelte-trusted-html" and only assigns templates that were compiled from .svelte files at build time. No user input, no YouTube content, and no third-party data ever flows to innerHTML. You can confirm: `grep -r "innerHTML\|@html\|{@html" src/` returns zero matches.
2. "data_collection_permissions requires Firefox 140+/Android 142+": informational. The manifest declares this key (set to "none" -- we collect nothing) so newer Firefox shows the no-data-collected badge. Older Firefox silently ignores unknown manifest keys; the extension still works on Firefox 109+ as declared by strict_min_version.

How to test
1. Install and open any YouTube watch page that has captions, e.g. https://www.youtube.com/watch?v=YwZR6tc7qYg
2. Open the Quoth sidebar (View > Sidebar > Quoth, or click the toolbar icon)
3. The transcript loads after the YouTube page finishes loading. Click any word to seek the video to that timestamp.

Privacy & permissions
- All processing is client-side. No analytics, no telemetry, no remote storage. Captions are fetched directly from YouTube's own caption API and cached in extension storage keyed by video ID.
- host_permissions is scoped to youtube.com only. No other site is accessed.
- Privacy notice: https://github.com/tednaleid/quoth/blob/main/docs/spec/privacy.md

Happy to clarify anything; the GitHub issues link is in the support URL field.
```
